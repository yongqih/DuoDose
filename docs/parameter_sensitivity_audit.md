# Parameter-sensitivity audit

## Scope and conclusion

The completed `cline-ch` sensitivity output contains 27 rows: three seeds,
three semi-real size factors, and three expected rates. These rows were
generated by nine RF fits, not 27 independent fits. All expected combinations
are present and successful, and the saved summary exactly reproduces the raw
means, sample standard deviations, minima, and maxima.

The expected-doublet-rate analysis is valid as a threshold/rank operating-point
analysis. The completed semi-real-size analysis is **not an isolated training
size analysis**: its historical runner scaled held-out test doublets as well as
fit/calibration doublets. Consequently, the saved size-factor comparison must
not be described as evidence that training size alone improved performance.

## Expected doublet rate dependency trace

| Configuration | Function | Operation affected | Output affected |
| --- | --- | --- | --- |
| `parameter_sensitivity.expected_doublet_rates` | `run_parameter_sensitivity.main` | Iterates rates after one seed/factor RF fit | Adds three rows sharing one continuous score vector |
| rate loop value | `_candidate_fpr` / `deterministic_top_fraction` | Selects exactly `round(rate * n_test_cells)` top scores | `high_RNA_singlet_FPR_at_expected_rate` and reconstructed rank threshold |
| `prediction.expected_doublet_rate = 0.08` | public `DuoDose.predict` -> `resolve_threshold` | Computes the public expected-rate threshold | Public binary and subtype calls only |

The sensitivity rate is not passed into semi-real construction,
`run_protocol_models`, SafeFeature fitting, RF fitting, sample weighting,
calibration, probability prediction, continuous overall score, or conditional
subtype probabilities. Identical AUROC/AUPRC values across 0.05, 0.10, and
0.15 are expected and are not evidence that the rate was accidentally ignored.

## Semi-real size dependency trace

The completed runner multiplied all six configured homotypic/heterotypic
train, validation, and test count keys. `run_protocol_models` did not consume
the explicit validation keys; validation counts were instead derived as 25%
of the scaled `n_train_*` values. The effective historical counts per subtype
were therefore:

| Factor | RF fit doublets | Calibration doublets | Test doublets |
| --- | ---: | ---: | ---: |
| 0.5 | 188 | 62 | 125 |
| 1.0 | 375 | 125 | 250 |
| 2.0 | 750 | 250 | 500 |

The size factor changed RF fit sample amount, training class balance relative
to retained singlets, calibration sample amount, test doublet amount, test
doublet identities, test parent identities, and test doublet prevalence. RF
hyperparameters, `class_weight="balanced_subsample"`, random seed, genes,
feature schema, cluster definition, reference singlet pool, fitted transformer,
and high-RNA singlet identities remained fixed within each seed.

The future sensitivity runner now uses explicit fixed validation counts and
scales only fit-split homotypic and heterotypic counts. Default construction
behavior outside this sensitivity path is unchanged.

## Evaluation and reference fingerprints

Targeted reconstruction of the nine historical fits showed:

- validation and test cell-ID hashes differ across factors;
- validation and test parent-ID hashes differ across factors;
- high-RNA test-singlet ID hashes are identical within each seed;
- reference-pool ID hashes are identical within each seed;
- `safe_feature_reference_pool_id` and `safe_feature_transformer_id` are
  identical within each seed;
- feature schema, cluster definition, and gene count are identical within each
  seed.

Thus the fitted reference and transformer were stable, but the complete held-
out evaluation set was not.

## High-RNA FPR trends

Both high-RNA FPRs are rank-based. The completed design produced these mean
budgets across seeds:

| Factor | Test cells | True-doublet K | Canonical selected fraction | Canonical FPR |
| --- | ---: | ---: | ---: | ---: |
| 0.5 | 1841 | 250 | 0.1358 | 0.3655 |
| 1.0 | 2091 | 500 | 0.2391 | 0.4731 |
| 2.0 | 2591 | 1000 | 0.3860 | 0.5885 |

The canonical FPR increases because its test-truth-derived selection fraction
nearly triples. In contrast, the expected-rate metric keeps a requested 5%,
10%, or 15% rank fraction. At 5%, its mean FPR falls from 0.1117 to 0.0872 to
0.0407. The corresponding mean thresholds rise from 0.6703 to 0.7880 to
0.8478. This is consistent with stronger separation at the extreme top ranks,
but it is confounded by changing test doublet composition and cannot be
attributed to training size alone.

The canonical metric is not a fixed-threshold calibration diagnostic. Score
medians shift with size, but the opposite FPR trends are principally explained
by different rank budgets and changing test prevalence, not by a 0.5 threshold
versus rank threshold distinction.

Mean overall AUPRC rises from 0.5050 to 0.6465 to 0.7572 while the positive
test prevalence rises from 13.6% to 23.9% to 38.6%. Mean AUROC is 0.8053,
0.8167, and 0.8160. The completed output therefore does not establish a
progressive ranking improvement from larger training data.

## Parent and duplicate integrity at 2x

Across all three 2x reconstructions, every requested validation-suite parent
and duplicate check was zero:

- train/validation/test parent overlaps;
- reference-parent overlap;
- retained-singlet parent overlap;
- raw, reversed, and canonical duplicate parent pairs;
- duplicate parent-map rows;
- duplicate generated cell IDs;
- duplicate generated expression profiles;
- cross-split canonical-pair and parent overlap.

The 2x result did not gain performance through duplicated pairs, duplicate
profiles, reference leakage, or cross-split parent reuse.

## Frozen default

The frozen protocol uses `expected_doublet_rate = 0.08`. A size factor is not
a public model parameter; `semi_real_size_factor = 1.0` denotes the unscaled
frozen counts (500 requested training homotypic and 500 heterotypic doublets,
with the historical 375/125 fit/calibration split per subtype). The protocol
sets `select_configuration_on_test_data: false`, and its hash is recorded in
the run manifest.

The 1x setting remains the defensible default because it is the prespecified
unscaled computational setting and balances sample amount with cost. It was
not chosen as the empirical winner of this test-set sensitivity output. The
completed historical grid should not be used for post-hoc model selection.

## Recommended manuscript wording

> Expected doublet rate changed only the rank-based operating point and did
> not alter model fitting or continuous DuoDose scores. In the initial
> semi-real size sensitivity analysis, larger size factors also increased the
> held-out constructed-doublet count and prevalence; those results therefore do
> not isolate training-set size. The prespecified 1x setting was retained, and
> a corrected train-only sensitivity analysis is required before making a
> training-size robustness claim.

For the final manuscript benchmark, describe `high_RNA_singlet_FPR` as the
high-RNA false-positive fraction at matched 50% homotypic recall. Fixed top-20%,
matched 70%/80%, and the historical true-controlled-doublet budget are
supplementary operating-point analyses. Parameter-sensitivity outputs that use
expected-rate rank budgets must remain explicitly named and must not be mixed
with the primary matched-recall metric.
